# Mall Customer Segmentation

An unsupervised machine learning project that segments mall customers into distinct groups based on their Annual Income and Spending Score using KMeans clustering.

---

## Cluster Segments

| Cluster | Profile | Strategy |
|---|---|---|
| 1 | Low Income, Low Spend | Budget promotions |
| 2 | High Income, Low Spend | Premium product awareness |
| 3 | Mid Income, Mid Spend | Loyalty programmes |
| 4 | Low Income, High Spend | Instalment / EMI offers |
| 5 | High Income, High Spend | VIP / exclusive deals |

## Dataset

**Mall_Customers.csv** — 200 mall customers with 5 columns:

| Column | Description |
|---|---|
| CustomerID | Unique customer ID |
| Gender | Male / Female |
| Age | Customer age |
| Annual Income (k$) | Annual income in thousands of dollars |
| Spending Score (1-100) | Score assigned by the mall based on spending behavior |

Only `Annual Income` and `Spending Score` are used as features.

Download from [Kaggle — Mall Customer Segmentation](https://www.kaggle.com/datasets/vjchoudhary7/customer-segmentation-tutorial-in-python).

## Pipeline

| Step | Details |
|---|---|
| Load | Reads CSV, checks for missing values |
| Features | Selects `Annual Income` and `Spending Score` only |
| Elbow | Runs KMeans for k=1 to 10, plots WCSS to find optimal k |
| Train | KMeans with `n_clusters=5`, `n_init=10`, `random_state=42` |
| Visualise | Scatter plot of all 5 segments with centroids marked |
| Predict | Classifies new customers into one of 5 segments |
| Save | Pickles trained model to `models/kmeans_model.pkl` |

---

## Key Design Decisions

- **Only 2 features used** — Age and Gender are excluded; income and spending score are the most meaningful dimensions for marketing segmentation
- **Elbow method** — WCSS plotted for k=1..10 to find the natural "elbow" at k=5 rather than guessing
- **`n_init=10`** — runs KMeans 10 times with different centroid seeds and picks the best result; avoids poor local minima
- **`random_state=42`** — ensures reproducible cluster assignments across runs
- **Realistic test cases** — sample predictions use values within the dataset range (income 15–137k$, spending 1–99)
